# The Programmer Is No Longer Human: Engineering Framework for Large Scale Agent-Driven Software Development

## 大纲（Outline）

---

## Section 1: Introduction

- 现象：AI coding 工具已经能产出生产级代码，但团队实际效果方差极大
- 观察：差异不来自模型能力，而来自如何组织人与 Agent 的工作
- 核心洞察：软件工程的制度体系是围绕人类执行者设计的，当执行者变成 AI Agent，制度需要重建
- 本文贡献：识别 AI Agent 的五个结构性特征，推导出新条件下的工程挑战，提出对应的制度设计框架
- **Takeaway: 执行者变了，制度必须跟着变**

---

## Section 2: Problem Scope

- 软件交付的核心任务
- 执行者在其中的角色
- 本文范围：产品工程，中大型项目
- **Takeaway: 我们讨论的是"执行者变了"对软件交付的影响**

---

## Section 3: 新执行者 — AI Agent 的五个结构性特征

- 3.1 高吞吐零边际成本
- 3.2 忠实执行
- 3.3 有限处理容量
- 3.4 无记忆积累
- 3.5 无后果感知
- 每个特征内带出与人类的对比
- 高吞吐的特殊角色：放大其他四个
- **Takeaway: AI Agent 与人类执行者存在五个结构性差异**

---

## Section 4: Challenges

- 4.1 产出速度超过人类质量保障带宽（← 高吞吐）
- 4.2 规约模糊点变成随机决策（← 忠实执行）
- 4.3 未提供的知识不会被补全（← 忠实执行）
- 4.4 任务规模与质量负相关（← 有限处理容量）
- 4.5 长链推理中早期决策被遗忘（← 有限处理容量）
- 4.6 组织知识不会自然沉淀和演化（← 无记忆）
- 4.7 没有内在质量驱动力（← 无后果感知）
- 4.8 不区分风险等级（← 无后果感知）
- 每条指出传统方法为什么在此失效
- **Takeaway: 新条件下传统方法无法应对**

---

## Section 5: Framework

### 5.1 自动化验证体系（← 4.1）

- 测试前置
- 对抗式 AI review
- 人类角色从审查代码转为审查测试和规约

### 5.2 规约体系（← 4.2, 4.3）

- 机器可读规约，验收标准即评估函数
- 文档测试
- 上下文工程
- 信任环

### 5.3 任务分解（← 4.4, 4.5）

- 模块化，API Contract
- 小步推进，每步验收后重新整理上下文
- 敏捷需求瀑布执行
- 执行时上下文扁平化

### 5.4 知识沉淀机制（← 4.6）

- 技能卡作为跨会话知识载体
- 双环学习

### 5.5 外部质量治理（← 4.7, 4.8）

- 对抗式验证
- 测试锚定人工验收标准
- 奖杯测试
- 信任边界

- **Takeaway: 用显式制度设计替代人类执行者自带的天然能力**

## Section 6: Discussion

- 特征会变吗：模型进步如何改变五个特征的强度，框架如何随之演化
- Challenge 的交互：单特征分析的简化与实践中的叠加效应
- 框架的边界：什么类型的项目或任务不适用，推导链条在哪里断裂
- **Takeaway: 框架是方法论而不是静态清单，特征变化时沿同样链条重新推导**
